Predicting Malicious URLs

Background¶

Traditional firewall relies on whitelist and blacklist to decide whether to allow or block a domain name. This scheme is problematic as the lists need to be updated manually and regularly. Outdated lists would limit the accessibility of the network and/or fail to detect novel malicious domain names.

So we aim to train a machine learning model to predict whether a domain name is malicious. The model can be fully developped into a smart firewall solution to police network traffic without relying on static lists.

Problem

Given a url, such as google.com or pakistanifacebookforever.com, predict whether it is malicious or benign.

Solution

The basic idea is that we will dump a list of both malicious and benign urls at a machine learning model. The model would be tuned to correctly identify good and bad urls.

We will use an available dataset of malicious and benign urls [https://goo.gl/Lqi9fQ].

Each url is transformed to a bag of word representation [https://en.wikipedia.org/wiki/Tf%E2%80%93idf].

Then, we train a couple of models to do the prediction: 3-layer Neural Network, Logistics Regression and Random Forest.

Results

The best result is achieved by Logistics Regression (97.80% accuracy) on the holdout validation set. It's often the case for ill-posed problems with high-dimensional and sparse feature space that simple models works the best. Simple models usually have the lowest training time also. The following table shows the accuracry and the training time of all models.

Extract the features¶

We will use the bag-of-word model to represent each url. Three delimiters /, ., - are used to split each url into a set of tokens. Each url is represented by the frequencies of its set of tokens.

By this way, the tokens with high frequency in the whole dataset would not be good signals for the labels of the urls. This bring the necessarity of the last step: normalizing the features using the the appearance frequency in the whole dataset.

In the end, the formula to compute the value for each feature is:

$$w_{i,j} = tf_{i,j} \times \log{\frac{N}{df_j}}$$

In this formula, $w_{i,j}$ is the value of token $j$ in url $i$, $tf_{i,j}$ is the frequency of token $j$ in url $i$ and $df_j$ is the frequency of token $j$ in the whole dataset.

# FUNCTION TO EXTRACT TOKENS FROM URL
def extract_tokens(url):
    tokens_by_slash = url.replace('-','/').split('/')
    all_tokens = tokens_by_slash
    
    for slash_token in tokens_by_slash:
        tokens_by_dot = slash_token.split('.')
        all_tokens = all_tokens + tokens_by_dot
        
    all_tokens = list(set(all_tokens))
    
    if 'com' in all_tokens:
        all_tokens.remove('com')
        
    if '' in all_tokens:
        all_tokens.remove('')
        
    return all_tokens

# FUNCTION TO NORMALIZE THE WEIGHTS
def norm_weight(url_freq, doc_freq, total_doc):
    return url_freq * np.log(float(total_doc)/doc_freq)

# EXTRACT ALL POSSIBLE TOKENS AND COUNT THEIR FREQUENCY
all_urls = []
all_tokens = {}

for url in df['url']:
    tokens = extract_tokens(url)
    all_urls.append(tokens)
    
    for token in tokens:
        if token in all_tokens:
            all_tokens[token] += 1
        else:
            all_tokens[token] = 1
        
token_2_id = dict(zip(all_tokens.keys(), range(len(all_tokens))))

# CONSTRUCT THE FINAL DATASET FOR TRAINING
url_num = len(all_urls)
entries = [(norm_weight(1., all_tokens[token], url_num), i, token_2_id[token]) for i in xrange(url_num) for token in all_urls[i]]
entries = np.array(entries)

data    = entries[:,0]
row_ind = entries[:,1]

3-layer neural network¶

In this section, we will use a 1 hidden layer neural network to predict malicious urls. The hidden layer contains 32 nodes. We will use early stopping to stop the training at the moment when the validation loss does not get improved.

inputs = Input(shape=(X_train.shape[1],))

x = Dense(32, activation='relu')(inputs)
x = Dropout(0.5)(x)
outputs = Dense(1, activation='sigmoid')(x)

model = Model(inputs=inputs, outputs=outputs)

Because the unwrapped dataset is too big, it cannot be fitted in to RAM. We need to use a generator to unwrap the dataset batch by batch.

def generate_batch(train, label, batch_size):
    num_exm = train.shape[0]
    
    keys = range(num_exm)
    np.random.shuffle(keys)
    
    l_bound = 0
    r_bound = batch_size if batch_size < num_exm else num_exm
    
    while True:
        if l_bound == r_bound:
            l_bound  = 0
            r_bound = batch_size if batch_size < num_exm else num_exm
            np.random.shuffle(keys)
        
        x_batch = train[keys[l_bound:r_bound]]
        y_batch = label[keys[l_bound:r_bound]]
        
        yield x_batch.toarray(), y_batch
        
        l_bound = r_bound
        r_bound = r_bound + batch_size
        if r_bound > num_exm: r_bound = num_exm    

early_stop  = EarlyStopping(monitor='val_loss', min_delta=0.001, patience=10, mode='min', verbose=1)
checkpoint  = ModelCheckpoint('weights.hdf5', monitor='val_loss', verbose=1, save_best_only=True, mode='min', period=1)
tensorboard = TensorBoard(log_dir='../logs/', histogram_freq=0, write_graph=True, write_images=False)

all_exams  = X_train.shape[0]
trv_split  = int(0.9*all_exams)
batch_size = 8
indices    = np.random.permutation(Y_train.shape[0])

train_gen = generate_batch(X_train[indices[:trv_split]], Y_train[indices[:trv_split]], BATCH_SIZE)
valid_gen = generate_batch(X_train[indices[trv_split:all_exams]], Y_train[indices[trv_split:all_exams]], BATCH_SIZE)

train_num = int(np.ceil(trv_split/batch_size))
valid_num = int(np.ceil((all_exams - trv_split)/batch_size))

minimizer  = SGD(lr=0.0001)
model.compile(optimizer=minimizer, loss='binary_crossentropy', metrics=['accuracy'])
model.fit_generator(generator = train_gen, 
                    steps_per_epoch = train_num, 
                    epochs = 100, 
                    verbose = 1, 
                    validation_data = valid_gen, 
                    validation_steps = valid_num, 
                    callbacks=[early_stop, checkpoint, tensorboard],
                    max_q_size = 3)
